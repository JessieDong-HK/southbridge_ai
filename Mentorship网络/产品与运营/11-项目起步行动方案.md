# 11 · 项目起步行动方案

> 2026-06-18，基于四家平台调研结论。目标：**不做大而全的规划，只回答"接下来几周具体干什么能让项目跑出第一次交付"。**

---

## 调研告诉我们的五件事

1. **供给端先行**——四家全部是先攒 mentor，需求跟着 mentor 的 network 自然来
2. **不花钱投广告**——四家早期全是 organic。Mentor profile 本身 = 最好的广告
3. **从单一品类切入**——Leland 只做 MBA → 扩展到所有品类。我们先只做美本转学
4. **最简单的 MVP 先跑**——ADPList 用 Google Sheet 起步。我们不需要小程序 ready 才能开始
5. **耐心**——MentorCruise 头 2.5 年几乎没增长。别急

---

## 第一阶段目标（2-3 周）

> **完成第一次交付：一场美本转学分享会（产品 1，¥99/599），有人报名、有人讲、有人听完觉得值。**

不是"建完平台"、不是"确认完所有 mentor"、不是"自媒体做起来"——就是**一次真实的交付**。

---

## 四条并行线

```
        确认首批 mentor        写完整 profile
        （3-5人，转学赛道）    （用采访提纲深挖 + 标签打标）
              │                    │
              └────────┬───────────┘
                       │
                       ▼
              最简单版"平台"上线
              （可以只是一个飞书文档 / 石墨页面 /
               或者 Echo 搭的一个极简 landing page）
                       │
                       ▼
              第一场分享会落地
              （定 mentor + 定时间 + 定价 +
               发朋友圈/小红书招募学生）
                       │
                       ▼
              会后素材 → 自媒体第一波内容
```

---

## 线 A｜确认首批 mentor（3-5 人）

**目标**：从 15 人池子里，确认 3-5 个**美本转学赛道**的 mentor 愿意正式参与。

**具体动作**：

| # | 动作 | 说明 |
|---|------|------|
| A1 | 从 6 位转学生（素心/Mia/Jas/Claire/张祺然/Gabrielle）中优先联系 | 他们已经有 coffee chat 基础，不需要从头解释项目 |
| A2 | 发项目介绍（PPT/PDF） | 用 [05-Mentor招募材料-项目介绍框架.md](05-Mentor招募材料-项目介绍框架.md) 做一版能发出去的 |
| A3 | 确认每个人的：可用时间、愿意做的服务层级、对外展示偏好（实名/匿名） | 尤其注意素心（休学中）和 Alice（标注不可对外） |
| A4 | 每人安排一次深度采访 | 用 [06-Mentor深度采访提纲.md](06-Mentor深度采访提纲.md)，方式 A 或 B |

**为什么只要 3-5 人**：四家平台起步时 mentor 都不超过 15 个。先跑通一个人完整的"采访→profile→上线→接单"流程，比同时处理 15 个人重要。

---

## 线 B｜写出第一批完整 mentor profile

**目标**：每个确认的 mentor 有一份可以直接对外展示的 profile（故事 + 标签 + 可约服务）。

**具体动作**：

| # | 动作 | 说明 |
|---|------|------|
| B1 | 用采访回答 + 标签勾选 → 润色为完整 profile | 叙事逻辑参考 [06-Mentor深度采访提纲.md](06-Mentor深度采访提纲.md) 里的"从回答到 Profile 的转化思路" |
| B2 | 每个 profile 包含：路径摘要 + 完整故事 + 能力标签 + 活人标签 + 可约层级 | 参考 [07-Mentor分级标签系统.md](07-Mentor分级标签系统.md) |
| B3 | 给 mentor 确认并修改 | 对方有修改权——自己认可了才愿意发 |

---

## 线 C｜搭建最简单版展示 + 预约入口

**目标**：有一个地方能让潜在学生看到 mentor、了解他们、并且表达报名意向。**不需要小程序 ready。**

**方案选项**（从简到繁）：

| 方案 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| C1｜飞书/石墨文档 | 一个精美的文档页：mentor 列表 + profile + 报名表链接 | 今天就能做，零开发 | 不够"产品感" |
| C2｜Echo 搭极简 landing page | 一个单页网站：mentor 卡片 + 报名/付款入口 | 有点产品感，开发量小 | 需要等 Echo 有空 |
| C3｜微信群 + 公众号文章 | 公众号发 mentor 介绍文 → 评论区/私信报名 | 在微信生态内，传播方便 | 长期不好管理 |

**建议**：**C1 + C3 并行先跑**。飞书文档做 mentor 展示（反正第一批就 3-5 人），公众号文章做传播。Echo 的小程序/网站同步开发，不急。

**具体动作**：

| # | 动作 | 说明 |
|---|------|------|
| C4 | 做一个精美的 mentor 展示页（飞书/石墨） | 包含：赛道入口 → mentor 卡片 → 完整 profile → 报名入口 |
| C5 | 设置报名/付款流程（MVP：微信群 + 微信转账） | 先不接支付系统——微信转账 + 手动确认最快 |
| C6 | 跟 Echo 同步最新需求（小程序优先 + 运营自动化链路） | 参考 [08-平台运营流程设计.md](08-平台运营流程设计.md) |

---

## 线 D｜落地第一场分享会

**目标**：有人报名、有人讲、有人听完觉得值。产出第一批自媒体素材。

**具体动作**：

| # | 动作 | 说明 |
|---|------|------|
| D1 | 定一位 mentor 做第一场分享 | 优先选表达好、故事强、时间灵活的（Gabrielle 或 Mia 可能合适） |
| D2 | 定时间 + 定价（¥99 / ¥599 两档） | 线上，1-1.5h |
| D3 | 帮 mentor 准备分享提纲 | 参考产品 1 的 timeline walkthrough 结构 |
| D4 | 发招募信息 | Jess 朋友圈 + 小红书 + 相关微信群 |
| D5 | 执行分享会 | 腾讯会议，录屏（经 mentor 同意） |
| D6 | 会后：收集反馈 + 剪辑素材 | 一场分享会拆 5-10 条自媒体内容 |

---

## 优先级排序

```
本周必须动：
  A1 联系首批 mentor          ← 没有 mentor 什么都别谈
  C4 做 mentor 展示页          ← 飞书文档，今天就能动手

本周尽量动：
  A4 安排第一场深度采访        ← 产出第一个完整 profile
  D1 定第一场分享会的 mentor   ← 产品 1 的最简版本

下周：
  B1-3 写出并确认 profile
  C5 报名/付款流程
  D3-4 分享会准备 + 招募

Echo 层面（不阻塞主线）：
  C6 同步小程序需求
```

---

## 不做的（现在不做）

- ❌ 完善所有 15 人的 profile
- ❌ 建完所有赛道的能力标签
- ❌ 搭完整小程序/网站
- ❌ 设计长线陪跑套餐的运营流程
- ❌ 自媒体全平台矩阵
- ❌ B2B / 机构合作
