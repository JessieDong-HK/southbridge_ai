# 09 · 标签系统重设计方案 · 讨论稿

> 背景：现有 60+ 二级标签系统是「服务视角」（我们能提供什么），需要改成「学生视角」（我有什么问题、谁能帮我）。

---

## 一、现有系统的问题（坦诚版）

| 问题 | 具体表现 |
|------|---------|
| **视角反了** | 「学业规划」「科研背景提升」是运营者分类，学生不会这么搜 |
| **太多了** | 60+ 个二级标签，迷茫的学生面对 60 个选项只会关掉 |
| **赛道被弱化** | 「谁走过我想走的路」是学生最直觉的筛选，但赛道标签埋在一堆标签里 |
| **缺少「什么人」** | 没有风格/气质维度——学生选 mentor 不只是选技能，是选「我想和谁聊」 |
| **和产品脱节** | 标签系统和产品层级（¥99分享会→¥3万全包）没有对应关系 |

---

## 二、学生视角的三个决策阶段

模拟一个学生打开小程序的完整心理路径：

```
阶段1：「谁走过我想走的路？」
  → 浏览这个赛道下面的 mentor
  → 看到每个人的路径摘要（from → to）
  → 产生代入感："这个人的背景和我好像"

阶段2：「谁能解决我现在的具体问题？」
  → 在赛道内进一步缩小范围
  → 按痛点筛选：我要的是文书指导？还是全局规划？还是心态支持？

阶段3：「这个人我想聊」
  → 点进 profile 看故事
  → 感受这个人的风格、气质、说话方式
  → 决定：我要约这个人
```

**现有的标签系统只覆盖了阶段 2，而且覆盖得很不好。阶段 1 和阶段 3 基本缺失。**

---

## 三、新结构提议

### 三层体系（对应三个阶段）

```
第一层｜赛道标签（阶段1：谁走过我的路）
  ── 学生在首页第一眼看到的就是赛道入口
  ── 单选。选完后进入该赛道的 mentor 列表

第二层｜痛点标签（阶段2：我需要什么帮助）
  ── 学生语言，不是行业术语
  ── 多选。每个赛道下的痛点不同

第三层｜风格标签（阶段3：这个人什么感觉）
  ── 不是筛选条件，是 profile 上的展示元素
  ── 帮学生在浏览时快速感知"这个人我聊不聊得来"
```

---

### 第一层：赛道标签

> 学生心理：「我想做你做过的事」

| 赛道 | 对应学生 |
|------|---------|
| 🔥 美本转学 | 我在美国读本科，想转到更好的学校 |
| 美本申请 | 我在高中，想申请美国本科 |
| 美研申请 | 我想申请美国硕士/博士 |
| 英联邦申请 | 我想申请英国/香港/新加坡/澳洲 |
| 高考→海外 | 我在普高，想走非传统路线出国 |
| 国内升学 | 我想保研/考研到国内名校 |

**设计问题讨论**：
- 赛道要不要更细？（比如把「美本转学」拆成「美本→美本」「陆本→美本」「社区大学→大U」？）
- 还是保持大类，细分的匹配靠 mentor profile 里的路径叙述来解决？

---

### 第二层：痛点标签

> 学生心理：「我现在卡在哪里」

**这里的关键问题**：痛点标签是所有赛道通用的，还是每个赛道有一套自己的？

**方案 A｜通用痛点（所有赛道共用一套）**

| 痛点 | 学生语言 |
|------|---------|
| 🎯 全局规划 | "我完全不知道从哪里开始，需要人帮我理清方向和时间线" |
| ✍️ 文书与材料 | "我的文书/CV/活动列表需要指导" |
| 📚 学术背景 | "我 GPA 不够/不知道怎么选课/找不到科研机会" |
| 🏫 选校定位 | "我不知道该申请哪些学校，哪些适合我" |
| 💬 面试准备 | "我需要 mock interview" |
| 🧠 心态与决策 | "我很焦虑/迷茫/不确定要不要走这条路" |

**方案 B｜赛道定制痛点（每个赛道有自己的痛点）**

以「美本转学」为例：
| 痛点 | 学生语言 |
|------|---------|
| 我不知道该不该转 | "转学值不值得？我会不会后悔？" |
| 我需要全局路线图 | "从现在到提交申请，每一步该做什么？" |
| 我的文书写不出来 | "Why transfer 到底怎么写？" |
| GPA 要崩了 | "选课踩坑，怎么救 GPA？" |
| 教授不理我 | "推荐信怎么办？Office Hour 聊什么？" |
| 我找不到科研 | "怎么找 RA？没有科研经历怎么办？" |
| 我快撑不住了 | "太累了，想找过来人聊聊" |

> **Jess 的判断**：选 A 还是 B？或者 A（通用）+ B（每个赛道补充 2-3 个特有痛点）？

---

### 第三层：风格标签

> 学生心理：「这个人的 vibe 适不适合我」

**不是 filter（不用来筛选），是 profile 上展示的标签。** 目的是让学生快速感知 mentor 的气质，降低"不知道该选谁"的 friction。

| 风格 | 含义 |
|------|------|
| 🔥 直球型 | 直接告诉你问题在哪，不绕弯子 |
| 🌸 温暖陪伴型 | 先接住你的情绪，再给建议 |
| 📊 数据/逻辑派 | 用事实说话，每一步都有依据 |
| ☕ 轻松聊天型 | 像朋友聊天，不像上课 |
| 🎯 结果导向型 | 一切都是为了拿到 offer |
| 🌱 成长导向型 | 重视你这个人长期的变化，不只是申请结果 |

**设计问题讨论**：
- 风格标签由谁定？Mentor 自选 vs Jess 根据采访判断？
- 要不要限制每个 mentor 最多选 1-2 个？（太多 = 没有风格）

---

## 四、和产品层级的关系

还有一个被忽略的维度：**这个 mentor 提供什么级别的服务？**

目前产品有 4 级（分享会 / Kickstart / 长期 / Elite），但 mentor 不一定每级都做。

**提议**：在 mentor profile 上展示他/她开放的产品级别，而不是作为标签让学生筛选。比如：

```
Gabrielle 的 profile 页：
  赛道：美本转学
  路径：UNC → 芝加哥
  痛点：心态疏导 · 文书逻辑 · 选校调研
  风格：🌸 温暖陪伴型 · 🌱 成长导向型
  可约：🎤 分享会 · 💬 轻咨询 · 🌱 长期 mentor
```

| # | 问题 | 选项 |
|---|------|------|
| 1 | 赛道粒度：大类 vs 细分？ | A. 保持 6 个大类 B. 拆更细（如美本转学拆成 3 个子类） |
| 2 | 痛点标签：通用 vs 定制？ | A. 全平台一套通用痛点 B. 每个赛道自己的一套痛点 C. 通用 + 赛道特有 |
| 3 | 痛点最多选几个？ | 3? 5? 不限？ |
| 4 | 风格标签：自选 vs 我们定？ | A. Mentor 自选 B. Jess 根据采访判断 C. 两者结合 |
| 5 | 产品层级要不要做成标签？ | A. 展示在 profile 上但不做筛选 B. 也做成筛选条件 |
| 6 | 现有 60+ 二级标签怎么办？ | A. 全部废弃 B. 保留作为内部运营使用 C. 部分转化为学生语言 |
