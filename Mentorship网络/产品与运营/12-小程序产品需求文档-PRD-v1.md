## 一、项目概述

国际教育 Mentorship 网络——连接有真实经历的名校学长学姐和迷茫的学生。第一个切入点：美本转学。

### 用户角色
- **学生（Mentee）**：想走某条留学路但迷茫
- **Mentor**：走过这条路、拿到真实结果的人
- **管理员（Jess）**：策展 mentor、处理异常

### 平台形态
- 微信小程序（主）+ 网站（辅）

---

## 二、页面结构

```
1. 首页
2. Mentor 列表页
3. Mentor 详情页
4. 预约流程（4 步）
5. 支付成功页
6. 我的（学生端）
7. Mentor 端
8. 管理后台（Web）
```

## 三、各页面规格

### 1. 首页

**Hero**：标题「找到走过你想走的路的人」+ 副标题「真实的过来人经验，不是中介模板」

**赛道入口（2×3 grid）**：
- 6 张卡片，每张含 emoji + 标题 + 一行描述
- 🔥 美本转学为 primary 样式（浅紫背景 + 紫边框），其余灰色边框
- 点击进入该赛道的 Mentor 列表页

**精选 Mentor（横向滚动）**：
- 3-5 张 140px 宽卡片：圆形头像(48px) + 姓名 + 路径摘要

**如何参与（3 步）**：
- 三列等宽：紫色编号圆圈 + 简短文案
- ① 选赛道 → ② 看故事 → ③ 约时间

> 👀 见原型 Screen 1

### 2. Mentor 列表页

**导航**：`‹ 美本转学`

**Filter chips**（横向，多选）：全部 / 文书 / 选校 / 推荐信 / 背景提升 / 面试 / 心态 / 材料流程。默认「全部」active（紫底白字）。选中后列表实时筛选。

**排序**（横向文字，单选）：推荐（默认）/ 评分最高 / Session 最多

**结果数量**：「共 X 位 mentor」

**Mentor 卡片**（垂直排列）：

| 区域 | 内容 | 样式 |
|------|------|------|
| 顶部 | 圆形头像(52px) + 姓名(16px加粗) + 路径摘要(13px #555) + 亮点数据(12px #888) | 横向布局，头像左侧 |
| 能力标签 | icon+文字，最多展示 7-8 个 | 12px，浅灰底(#f3f4f6)，小圆角 |
| 可约层级 | 💬 轻咨询 · 📋 短期 · 🌱 长期 | 12px，#4F46E5 |
| 活人标签 | 2-5 个 | 10px #aaa，#fafafa 底，便签贴纸风 |

整张卡片可点击 → Mentor 详情页。

> 👀 见原型 Screen 2（素心、Gabrielle、Mia 三张真实卡片）

### 3. Mentor 详情页

**Header**（居中）：大头像(80px) + 姓名(22px) + 路径摘要(14px #555) + 亮点(13px #888)

**关于我**：16px 加粗标题 + 14px 正文(行高1.8)，500-800 字。由 Jess 团队撰写，叙事逻辑：moment 开头 → 成长轨迹 → 转折 → 干货 → 我能帮你

**我能帮你**：按能力域分组——
- 每组 13px 灰色大写标题 + 标签横向排列
- 标签：12px #3730A3 + #EEF2FF 底 + 8px 圆角
- 未勾选的能力域不展示

**可约服务**：3 种类型的卡片列表（仅展示该 mentor 开放的层级）——
- 左侧：服务名(15px 加粗) + 描述(12px #888)
- 右侧：价格(18px #4F46E5 加粗)
- 1.5px #eee 边框 + 12px 圆角

**活人标签**：黄色暖调(#fef3c7/#92400e)，14px 圆角，横向排列

**底部 CTA**：固定底部，白底 + 分割线 + 紫色全宽按钮「预约 XXX」

> 👀 见原型 Screen 3（素心完整 Profile）

### 4. 预约流程（4 步）

**Step 1 — 选服务类型**：
- 顶部显示「STEP 1 / 4」
- Radio 列表：每行圆按钮 + 服务名(15px) + 描述(12px #888) + 价格(18px #4F46E5)
- 选中态：蓝紫边框 + 浅紫背景
- 只显示该 mentor 开放的服务层级
- 底部「继续 →」

> 👀 见原型 Screen 4

**Step 2 — 选时间**：
- 「STEP 2 / 4」
- 时区选择器（下拉，默认 GMT+8）
- 文案提示「多选几个时间段，mentor 越容易匹配上」
- 按天分组（接下来 7 天），每天展示时间 chip 列表
- 时间 chip：白底灰边 → 选中态紫底白字
- 底部「已选 X 个时间段」
- 底部「继续 →」

> 👀 见原型 Screen 5

**Step 3 — 简述情况**：
- 「STEP 3 / 4」
- 文案提示「选填，这些信息会发给 mentor」
- 3 个引导问题 + 文本框(14px, 圆角12px, focus紫边框)：
  1. 你现在在哪个学校？读什么专业？
  2. 为什么想转学？最想聊什么？
  3. 你目前最大的困惑或焦虑是什么？
- 底部「继续 →」

> 👀 见原型 Screen 6

**Step 4 — 确认 & 支付**：
- 「STEP 4 / 4」
- 确认卡片(灰底#f9fafb, 14px圆角)：Mentor名 / 服务 / 金额(18px #4F46E5) / 选的时间段列表
- 黄色警告文字：⚠️ Mentor 会从这些时间里选一个确认。如果都不行，mentor 会通知你重新选择。通常 12 小时内回复。
- 底部「💳 微信支付 ¥XXX」

> 👀 见原型 Screen 7

### 5. 支付成功页

- ✅ 大图标
- 「预约已提交」
- 描述：已通知 XX mentor，ta 会从你选的 N 个时间段里挑一个确认
- 黄色提示框(#fefce8/#92400e)：说明后续流程——通常 X 小时内回复 / 确认后收到腾讯会议链接 / 超时未回复会人工跟进 / 在「我的预约」查看状态
- 「返回首页」按钮（outline 样式）

> 👀 见原型 Screen 8

### 6. 我的（学生端）

- 我的预约列表（状态：待 mentor 确认 / 已确认 / 已完成）
- 历史 session
- 设置

### 7. Mentor 端

- 收到预约 → 选择确认时间 OR 全拒
- 我的 session 列表
- 收入记录

### 8. 管理后台（Web，Jess 用）

- Mentor 池管理
- 订单管理（含异常高亮）
- 数据概览

## 四、标签系统（数据模型）

详见 [07-Mentor分级标签系统.md](07-Mentor分级标签系统.md)。以下为开发者需要的 enum/常量：

**维度一：赛道**（Mentor 多选，学生单选）
```
['美本转学', '美本申请', '美研申请', '英联邦申请', '高考→海外', '国内升学']
```

**维度二：能力标签**（Mentor 多选），按能力域分组：
```
选校定位:        ['🏫 选校方向建议', '📊 完整 school list', '🔍 深度 fit 分析']
文书:           ['💡 经历挖掘 + brainstorming', '🏗️ 文书框架搭建', '👀 给 feedback', '✍️ 深度修改 polishing', '🖊️ 直接写/改写']
推荐信 & 教授关系: ['🤝 教授关系建立', '📩 推荐人选择 & 时机', '✍️ 推荐信撰写']
申请材料 & 流程:  ['📋 材料准备 & 流程']
学术与背景提升:   ['📚 选课策略 & GPA', '🔬 找 research / RA', '🎨 活动规划 & 人设', '🏆 竞赛/夏校']
面试:           ['🎤 Mock interview', '🎯 面试策略']
心态与决策:      ['❓ 该不该走这条路', '🧠 长期陪伴 accountability']
```

**维度三：活人标签**（Mentor 自由填写，≤5 个，纯展示不筛选）

**维度四：可约服务**（Mentor 多选）
```
['💬 轻咨询', '📋 短期 mentor', '🌱 长期 mentor']
```
价格由平台统一制定（¥599 / ¥2,999 / ¥9,900）。

## 五、核心数据模型

### Mentor
```
{
  id, name, avatar, track[], path_summary, highlight,
  about, capabilities[], vibe_tags[], service_tiers[],
  is_active, session_count, rating, attendance_rate
}
```

### Student
```
{ id, wechat_openid, name, timezone, bookings[] }
```

### Booking
```
{
  id, student_id, mentor_id, service_tier, amount,
  status,  // pending_mentor_confirm | time_confirmed | completed | cancelled | escalated
  student_time_slots[], confirmed_time, meeting_link,
  student_note, mentor_no_response_since
}
```

## 六、MVP vs 完整版

| 功能 | MVP | 完整版 |
|------|-----|--------|
| 首页（赛道+精选+步骤） | ✅ | ✅ |
| Mentor 列表 + filter chips + 排序 | ✅ | ✅ |
| Mentor 详情页（故事+标签+服务） | ✅ | ✅ |
| 预约 4 步流程 | ✅ | ✅ |
| 微信支付 | ✅ | ✅ |
| 支付成功页 | ✅ | ✅ |
| Mentor 择一确认 | 微信群回复 | 小程序内操作 |
| 腾讯会议自动生成 | 手动建会 | API 对接 |
| 自动提醒(24h/2h/15min) | 手动发微信 | 微信通知推送 |
| 课后反馈 | 手动问 | 自动推送 |
| 管理后台 | 简化版 | 完整仪表盘 |
| 长线陪跑套餐 | ❌ | 待设计 |

**MVP 建议**：前 3-5 个 mentor 的 profile 可以 hardcode 或从 JSON 读取，先让展示和预约流程跑通。
