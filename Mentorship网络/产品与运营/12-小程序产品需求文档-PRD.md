## 一、项目概述

**国际教育 Mentorship 网络**——连接有真实经历的名校学长学姐和迷茫的学生。

- 第一个切入点：**美本转学**（后续扩展更多赛道）
- 不是中介，是陪伴式 mentorship
- 半公益、community-driven

### 用户角色

| 角色 | 描述 |
|------|------|
| **学生（Mentee）** | 想走某条留学路但迷茫，需要过来人指导 |
| **Mentor** | 走过这条路、拿到真实结果的学长学姐 |
| **管理员（Jess）** | 策展 mentor、处理异常、看数据 |

---

## 二、平台形态

| 形态 | 优先级 | 原因 |
|------|--------|------|
| **微信小程序** | 🔴 主 | 所有 mentor 和学生都在微信上，零摩擦 |
| 网站（Web） | 🟡 辅 | 有比没有好，非第一版必需 |

### 小程序技术要点
- 微信原生框架或 Taro/uni-app 等跨端框架
- 微信支付集成
- 需要微信服务号或企业微信（用于通知推送）
- 腾讯会议 API（后期接入，MVP 手动建会）

## 四、页面详细规格

### 4.1 首页

**Hero 区**：
- 标题：28px 加粗，「找到走过你想走的路的人」
- 副标题：15px #666，「真实的过来人经验，不是中介模板」

**赛道入口**（2×3 grid）：
- 6 张卡片，每张含 emoji + 标题 + 一行描述
- 🔥 美本转学 为 primary 样式（浅紫背景 #EEF2FF + 紫色边框 #4F46E5），其余为灰色边框
- 点击任一卡片 → 进入该赛道的 Mentor 列表页

**精选 Mentor 横滑**：
- 标题「✨ 精选 Mentor」
- 横向滚动，每张卡片 140px 宽：圆形头像(48px) + 姓名 + 路径摘要
- 3-5 位高评价 mentor

**如何参与**（3 步）：
- 标题「📋 如何参与」
- 3 列等宽：编号圆圈(#EEF2FF 底 + #4F46E5 字) + 简短标签
- ① 选赛道 → ② 看故事 → ③ 约时间

完整布局和配色见 [HTML 原型](../outputs/approved/小程序原型-v1-页面结构.html) Screen 1。

---

### 4.2 Mentor 列表页

**顶部导航**：`‹ 美本转学`（返回箭头 + 当前赛道名）

**Filter chips**（横向排列，可多选）：
- 全部 / 文书 / 选校 / 推荐信 / 背景提升 / 面试 / 心态 / 材料流程
- 默认「全部」选中
- 点击切换 active 态（白底灰边 → 紫底白字 #4F46E5）
- 选中后列表实时筛选

**排序**（横向文字，单选）：
- 推荐（默认）/ 评分最高 / Session 最多
- Active 态：#4F46E5 + 加粗

**结果数量**：「共 X 位 mentor」

**Mentor 卡片**（垂直排列，每张间距 12px）：
- **卡片顶部**（横向布局）：圆形头像(52px) + 姓名(16px加粗) + 路径摘要(13px #555) + 亮点数据(12px #888)
- **能力标签行**：最多展示前 7-8 个，icon+文字，浅灰底(#f3f4f6)小圆角标签
- **可约层级行**：💬 轻咨询 · 📋 短期 · 🌱 长期（#4F46E5 文字，12px）
- **活人标签行**：2-5 个浅灰小标签(#aaa 文字, #fafafa 底, 10px 字号)，像便签贴纸
- **交互**：整张卡片可点击 → 进入 Mentor 详情页

完整卡片样式见 [HTML 原型](../outputs/approved/小程序原型-v1-页面结构.html) Screen 2（含素心、Gabrielle、Mia 三张真实卡片）。

- 学生最少选 1 个，建议 ≥ 3 个（文案提示）
- 选中的时间段高亮

### 4.5 Mentor 端 — 预约确认

Mentor 收到微信通知后，打开小程序看到：

```
┌────────────────────────────────┐
│ 新的预约请求                     │
│                                │
│ 学生：XX 同学                    │
│ 服务：轻咨询（30min）             │
│ 情况：（学生填的简述）            │
│                                │
│ 请从以下时间里选一个：            │
│                                │
│ ○ 6/23 周一 14:00-15:00        │
│ ○ 6/24 周二 10:00-11:00        │
│ ○ 6/25 周三 15:00-16:00        │
│                                │
│ [确认这个时间]                   │
│                                │
│ ── 或 ──                       │
│                                │
│ [这些时间都不行]                  │
│                                │
└────────────────────────────────┘
```

**交互逻辑**（详见 [08-平台运营流程设计.md](08-平台运营流程设计.md) 模块 3）：
- Mentor 选一个时间 → 自动生成腾讯会议链接 → 通知双方
- Mentor 点"都不行" → 通知学生重新选时间
- Mentor 超过 X 小时不回复 → 通知 Jess 人工跟进

## 六、数据模型（核心字段）

### Mentor

```
{
  id: string
  name: string               // 中文名 + 英文名
  avatar: string             // 头像 URL
  track: string[]            // 赛道（维度一），如 ['美本转学', '美本申请']
  path_summary: string       // 路径摘要，如 "UC Davis → 宾大/康奈尔"
  highlight: string          // 亮点数据，如 "3.98 GPA · 5段科研"
  about: string              // 完整故事（800字以内，由Jess团队撰写）
  capabilities: string[]     // 能力标签（维度二）
  vibe_tags: string[]        // 活人标签（维度三），≤5个
  service_tiers: string[]    // 可约服务（维度四），如 ['💬', '📋', '🌱']
  is_active: boolean         // 是否展示（Jess可控制上下架）
  session_count: number      // 完成的 session 数
  rating: number             // 平均评分
  attendance_rate: number    // 出勤率
  created_at: datetime
}
```

### Student

```
{
  id: string
  wechat_openid: string      // 微信 openid
  name: string
  timezone: string           // 时区，如 'GMT+8'
  bookings: Booking[]
  created_at: datetime
}
```

### Booking

```
{
  id: string
  student_id: string
  mentor_id: string
  service_tier: string       // '💬' | '📋' | '🌱'
  amount: number             // 支付金额
  status: string             // 'pending_mentor_confirm' | 'time_confirmed' | 'completed' | 'cancelled' | 'escalated'
  student_time_slots: [      // 学生选的时间段（多个）
    { date: date, start: time, end: time }
  ]
  confirmed_time: datetime   // mentor 确认的时间（择一）
  meeting_link: string       // 腾讯会议链接（mentor确认后生成）
  student_note: string       // 学生填的简述
  mentor_no_response_since: datetime  // 用于升级判断
  created_at: datetime
}
```

## 八、开发建议

### 技术栈参考
- 小程序框架：微信原生 / Taro / uni-app
- 后端：Node.js / Python / Go（看开发者熟悉什么）
- 数据库：PostgreSQL 或 MySQL
- 文件存储：微信云开发 或 阿里云 OSS

### 开发顺序建议
1. **第一周**：Mentor 数据模型 + 赛道选择页 + Mentor 列表页 + 详情页（静态数据即可）
2. **第二周**：预约流程（选服务 → 选时间 → 简述 → 支付）+ 微信支付接入
3. **第三周**：学生端"我的"页面 + Mentor 确认（先做简单的，微信通知 + 手动确认）
4. **后续**：管理后台 + 通知系统 + 腾讯会议 API + 自动提醒

### 第一版可以用静态数据
前 3-5 个 mentor 的 profile 先 hardcode 或从 JSON 读取，不用搭后台。让展示和预约流程先跑通。
