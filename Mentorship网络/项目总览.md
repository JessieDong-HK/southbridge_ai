# 🧭 项目总览 · 国际教育 Mentorship 网络

> **一句话**：建立一个多切入点的国际教育 mentorship network——连接有真实经历的学长学姐和迷茫的学生，提供从轻咨询到长期陪伴的分层 mentorship 服务。**不是中介，而是一个半公益、community-driven 的成长网络。**

---

## 一、这个项目是什么

一个**多切入点的国际教育 peer mentorship 平台/社区**。学生可以在不同阶段、针对不同需求找到匹配的 mentor——已经走过这条路、拿到真实结果的学长学姐。

**核心定位**：
- ❌ 不是传统留学中介（代写文书、填表交材料）
- ✅ 是陪伴式 mentorship——mentor 用自己的真实经历帮助学生理解路径、建立方向、避免踩坑
- ❌ 不是只做转学
- ✅ 转学是第一个切入点（最容易启动），后续扩展到：美本申请、海外高中、高考→留学、陆本→美本/英本等更多赛道

**核心差异化**：
- mentor 是"走过这条路的人"，不是机构的销售顾问
- 陪伴式成长，不是代工式交付
- 社区归属感驱动，不是交易驱动

## 四、Mentor 社区

### 核心信念

> 我们希望 mentor 在这里获得的，不只是兼职收入，而是 **community、connection、成长、影响力**，以及真正帮助别人之后的成就感。

### Mentor 规模
- 已有 **50+ 导师**（英美港为主），尚未 active 推广
- 首批计划深度打造 **20 位核心 mentor** 的完整 profile

### Mentor 价值主张
1. **归属感**：聚集 genuinely driven 的人，定期线下 meetup / dinner / story sharing
2. **意义感**：收到真实的 mentee feedback，感受到自己真的帮助到了别人
3. **成长感**：通过 mentoring 练习 public speaking、storytelling、communication、leadership
4. **个人品牌**：mentor profile、spotlight、story feature、podcast 曝光

> 详见 [产品与运营/02-Mentor社区与文化.md](产品与运营/02-Mentor社区与文化.md)

## 八、MVP 网站蓝图（Jess · 2026-06-21 定调）

### 核心原则

> **第一版就是产品手册，做成网页版。看完这个网页，别人就知道这个项目是什么、为什么要购买你的服务、你和别人不一样的点在哪儿。**

**MVP 只做两件事：信息展示 + 意向收集。** 所有的预约、匹配、支付、跟进全部由人工完成——这不是技术不足，而是刻意为之。前期需要亲手跑通每一个环节，才能知道自动化应该替代什么。

**明确砍掉的功能（do things that don't scale）：**

| 砍掉 | 原因 | 后续 |
|------|------|------|
| ❌ 登录/注册 | 不需要用户系统 | V2+ |
| ❌ 在线预约/自动排期 | 前期人工匹配质量更高 | V2 |
| ❌ 在线支付 | 增加复杂度，拉长开发周期 | V2 |
| ❌ Mentor 入驻后台 | 前期 mentee 端都没跑通，mentor 端更不急 | V3 |
| ❌ 学生 Dashboard / 消息系统 | 纯展示阶段不需要 | V3+ |
| ❌ 小程序 | 先做网页版验证，小程序独立决策 | 独立评估 |

**唯一保留的交互**：预约意向表单（收集姓名+微信+有空时间段+想解决的问题）→ 提交后弹出小助手微信二维码 → 人工跟进。

### 信息架构

网站采用**单页滚动 + 独立表单页**结构：

| # | Section | 要回答的问题 | 内容要点 | 对应现有素材 |
|---|---------|------------|---------|------------|
| **Hero** | 这是什么？ | 一句话价值主张 + CTA 按钮 | "连接真实名校学长学姐的陪伴式 mentorship 网络——不是中介" | 项目总览一句定调 |
| **Why Us** | 和中介/竞品有什么区别？ | 陪伴式 vs 代工式、真实经历 vs 销售话术、社区归属感、半公益定位 | 项目总览 + 竞品调研（5 份报告） |
| **Mentor 展示** | 谁来带我？ | 3-5 位核心 mentor 的真实 profile：照片、学校、故事片段（从采访中提炼）、擅长领域标签 | 15人 mentor 池 + 标签系统 v3 + 深度访谈实录 |
| **产品 / 案例** | 我能得到什么？ | 四级产品体系简化展示 + 1-2 个真实辅导案例/学生反馈 | 产品体系设计 + 访谈实录 |
| **FAQ** | 还有疑问？ | 常见问题：价格、周期、和中介的区别、退款等 | 待整理 |
| **CTA 预约** | 怎么开始？ | 表单（姓名+微信+有空时段+想解决的问题）→ 提交后弹窗显示小助手微信二维码 | 平台运营流程设计 |

### 技术方案

| 维度 | 选择 | 说明 |
|------|------|------|
| 技术栈 | Next.js 16 + Tailwind CSS v4 + shadcn/ui | 复用 `ai-website-cloner-template` 脚手架，无需从零搭建 |
| 品牌规范 | 珊瑚色系、PingFang+Inter、Phosphor Icons、大圆角 | 见 `产品与运营/13-品牌识别规范.md` |
| 部署 | Vercel 免费层 | 自动 CI/CD，push 即部署 |
| 表单后端 | 先接飞书群 webhook / 轻量 Form API | 不需要自建后端，等量大了再升级 |
| 国际化 | 中英双语（至少首页） | 很多 mentor 是外国人，英文内容包容性很重要 |

### 给开发者的 Brief

```
搭建一个单页品牌官网，不是 SaaS 产品。
- 纯静态展示（Hero + Why Us + Mentor卡片 + 产品/案例 + FAQ + CTA）
- 唯一动态功能：预约意向表单（前端表单 → webhook → 飞书/微信群通知）
- 风格：温暖人文，不是冷冰冰的科技产品
- 参考：ADPList 的 mentor 卡片设计 + Leland 的叙事感
```
