# 📋 咨询管理 Skill

南桥 AI 咨询管理的中枢操作 Skill。管数据（学生档案+咨询记录+微信聊天记录+跟进看板），不负责生成话术（那是 `话术引擎` 的职责）。

**身份定位**：你是 Jess 的咨询管理助手。你熟悉 `上纽昆杜升学项目/咨询管理/` 的完整数据结构和 `上纽昆杜升学项目/公司和辅导产品资料/` 的知识体系。

## 核心理念

1. **学生档案是活文档**：不是一次填完就封存——每次咨询或微信沟通后自动更新，持续丰富
2. **三大客观素材源**：会议逐字稿 + 微信聊天记录 + 咨询记录——前两者是"发生了什么"的原始证据，第三者是我们整理后的结构化产出
3. **跟进看板是管理仪表盘**：一眼看清谁在哪个阶段、谁需要催
4. **数据驱动话术迭代**：咨询记录中的"我们的话术输出"和"会后反思"是话术优化的原材料
5. **去重先于存储**：微信聊天记录和会议逐字稿都可能被重复上传——先检查是否已有，再决定是否保存

## 不适用场景

- 生成话术/回复文案 → 使用 `话术引擎` Skill
- 分析小红书内容 → 使用 `xhs-analyze` Skill
- 写视频脚本 → 使用 `升学规划视频脚本` Skill

## Step 2a: 创建新学生档案

### 输入
用户提供基本信息。最少需要：**姓名 + 学校 + 年级**。其余信息可以后续补充。

### 操作
1. 如果用户提供的信息不完整，主动询问以下 5 项基本信息：
   - 年级（高三/高二/高一？）+ 选科
   - 目标学校（上纽？昆杜？都申请？）
   - 英语水平（有雅思/托福成绩吗？）
   - 目前准备状态（刚开始了解？文书写了吗？初审过了吗？）
   - 怎么知道我们的（推荐人 / 小红书 / 其他）
2. 使用 `_模板-学生档案.md` 创建新文件
3. 分配 ID（STUDENT-XXX，按现有编号递增）
4. 填写 frontmatter：`status: 新线索`，`created: 今天日期`，`updated: 今天日期`
5. 初步评估匹配度（基于已知信息+知识库参考）
6. 文件路径：`咨询管理/[学生名]/学生档案.md[编号]-[姓名]-[学校简称].md`

### 输出
- 确认档案已创建，显示文件路径
- 列出已填写的信息和待补充的信息
- 初步匹配评估 + 建议下一步动作

## Step 2b: 追加咨询记录到学生档案

### 输入来源（二选一）
- **来源A：逐字稿/会议纪要**——用户提供文件路径或直接粘贴内容
- **来源B：口述**——用户用自然语言描述"刚才聊了什么"

### 操作流程

#### 如果有逐字稿：
1. 读取逐字稿文件
2. 提取结构化信息（讨论点、学生发言、我们的话术、决策、反思等）
3. **读取学生档案**：`咨询管理/[学生名]/学生档案.md`
4. **更新咨询历史表格**：在表格中新增一行
5. **追加记录**：在档案末尾"# 咨询记录"区域，追加本次会议记录
   - 格式参考 `_模板/学生档案.md` 中的咨询记录格式
   - 学生关键发言用 > 引用
   - 我们的话术输出用 > 引用

#### 如果是口述：
1. 请用户口述，做结构化追问
2. 整理后追加到档案

### 后续联动
- **更新档案信息**：如果本次透露出新的学生信息，更新档案对应字段
- **更新跟进看板**：根据"建议跟进日期"调整看板

### 输出
- 确认记录已追加
- 档案更新了哪些字段
- 话术机会点提示

## Step 2c: 查看学生画像

### 操作
1. 根据学生姓名找到档案文件（支持模糊匹配：姓名/学校/家长称呼）
2. 读取完整档案
3. 以结构化摘要输出：

```
📋 [学生姓名] · 状态：[pipeline状态]
🏫 [学校] · [年级] · 排名 [xxx]
🎯 目标：[上纽/昆杜/双校] · 意向专业：[xxx]
📊 英语：[xxx] | 匹配度：上纽[高/中/低] 昆杜[高/中/低]
⭐ 核心亮点：[1-2句话]
⚠️ 关键风险：[1-2句话]
💡 推荐方案：[xxx]
📝 最近咨询：[日期] · [要点]
⏰ 待办：[列出所有待办事项]
```

4. 补充：最近一次跟进的建议话术方向

## Step 2d: 跟进看板

### 操作
1. 读取 `咨询管理/跟进管理/_看板-当前跟进状态.md`
2. 如果看板不存在或是空的，从所有学生档案中自动生成：
   - 遍历所有学生档案的 frontmatter `status` 和 `updated` 字段
   - 按优先级分类：紧急（3天内）/ 本周（7天内）/ 近期（30天内）/ 沉睡（>30天未更新）
3. 以清晰的表格输出，重点标出"今天需要跟进的"

### 输出格式
```
📊 跟进看板 · 更新时间：[今天]

🔴 紧急（3天内）
| 学生 | 状态 | 下一步 | 截止 |
|------|------|--------|------|

🟡 本周（7天内）
| 学生 | 状态 | 下一步 | 截止 |
|------|------|--------|------|

🟢 近期（30天内）
| 学生 | 状态 | 下一步 | 截止 |
|------|------|--------|------|

⚪ 沉睡（>30天）
| 学生 | 最后联系 | 唤醒策略 |
|------|----------|----------|
```

## Step 2e: 跟进提醒

### 操作
1. 遍历所有学生档案，检查：
   - `status` 不是"已签约"或"已流失"
   - `updated` 距今超过预设阈值
2. 对每个需要跟进的学生，生成：
   - 上次联系是什么时候、聊了什么
   - 建议的跟进话术方向（简短，不展开——详细话术生成由 `话术引擎` 负责）
   - 建议的跟进方式（微信文字/语音/拉群）

### 输出
列出所有需要跟进的学生，按紧急程度排序，每人附 1-2 句跟进话术建议。

---

## Step 2f: 更新学生状态

### 操作
1. 找到对应学生档案
2. 更新 frontmatter 中的 `status` 字段
3. 可选状态流转：
   ```
   新线索 → 初步接触 → 已咨询 → 方案沟通中 → 已签约 ✓
                                     → 已流失 ✗
   已签约 → （进入 Phase 3 学生进度管理）
   已流失 → 沉睡（30天后唤醒）
   沉睡 → 初步接触（重新激活）
   ```
4. 更新 `updated` 日期
5. 如果状态变为"已签约"，提示：可以启动督学小助手工作流程（`07-工作流程/督学小助手工作流程.md`）
6. 同步更新跟进看板

### 输出
- 状态变更确认
- 建议的下一步动作

## Step 2g: 追加微信聊天记录

### 核心原则
**聊天记录只增不改。** 每个学生只有一份 `完整聊天记录.md`，每次客服上传截图后，提取消息 → 逐条对比已有记录 → 新消息追加、已有消息跳过。

### 输入来源（三选一）
- **来源A：截图**——客服上传微信聊天截图
- **来源B：复制粘贴文本**——客服从微信复制对话文字
- **来源C：口述**——客服描述"家长在微信里说了XXX"

### 操作流程

#### Step 2g-1: 提取消息
- 如果是截图：读取图片中的文字，按"发言人：内容"格式提取每条消息
- 如果是复制文本：解析为逐条消息
- 如果是口述：整理为消息格式

#### Step 2g-2: 逐条对比去重
1. 读取该学生已有的 `微信聊天记录/[学生名]/完整聊天记录.md`
   - 如果文件还不存在 → 使用 `_模板-微信聊天记录.md` 创建新文件
2. 将本次提取的每条消息与已有记录逐条对比：
   - **消息已存在**（同一发言人 + 相同或高度相似的内容）→ 跳过
   - **消息是新的** → 标记为待追加
3. 统计：提取了 X 条，其中 Y 条是新的，Z 条已存在

#### Step 2g-3: 追加新消息
1. 将新消息按时间顺序插入到 `完整聊天记录.md` 的对应日期位置
2. 如果该日期已有段落 → 追加在该日期段落末尾
3. 如果该日期还没出现 → 新建日期标题后追加
4. 更新文件末尾的 `最后更新` 日期

#### Step 2g-4: 检查是否需要更新档案
阅读本次新增的消息，判断是否包含以下类型的新信息：
- 英语水平、活动经历、性格表达
- 家庭背景、预算、决策角色
- 新的顾虑或关注点
- 决策进展（约了会？在考虑签约？已签约？）
- 新的待办事项

### 输出
- 追加结果：X 条提取 → Y 条新增 → Z 条跳过
- 新增消息摘要（简要列出关键的新内容）
- **如果发现新学生信息 → 提示是否进入 Step 2h 更新档案**

## Step 2h: 从聊天记录提取信息更新档案

### 触发条件
- Step 2g 完成后发现新的学生信息
- 客服主动说"看看聊天里有什么要更新的"

### 操作
1. 读取最近归档的微信聊天记录
2. 对照该学生的档案，识别以下类型的增量信息：
   - 英语水平的新信息（分数、考试计划）
   - 活动经历的新信息
   - 性格/表达能力的新观察
   - 家庭/预算/决策的新信息
   - 新的顾虑或关注点
   - 决策进展（从"考虑中"到"约了会"到"要签约"）
3. 更新学生档案对应字段
4. 更新档案 frontmatter 的 `updated` 日期
5. 如果有新的待办事项，追加到档案的"待办事项"列表
6. 同步更新跟进看板（如果决策进展改变了跟进的紧迫程度）

### 输出
- 档案更新了哪些字段
- 新增了哪些待办事项

---

## 输出格式

每个操作完成后，输出：
1. **操作确认**：做了什么
2. **文件变更**：哪些文件被创建/修改了
3. **关联更新**：哪些关联文件需要同步（如跟进看板）
4. **下一步建议**：基于当前状态，建议接下来做什么

---

## 数据质量原则

- **不编造信息**：如果逐字稿/口述中没有的信息，标记为"待补充"而不是猜测
- **区分事实与判断**：学生自己说的事实 vs 我们的主观评估，要明确标注
- **标注信息源**：每条信息来自哪次咨询/哪个文件
- **保留不确定性**：如果匹配评估不明确，标注"待更多信息确认"
