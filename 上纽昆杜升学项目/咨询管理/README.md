# 咨询管理系统

> 南桥教育 AI 驱动的咨询线索与学生管理中枢。一个学生一个文件夹，所有资料归在一起。

## 客服标准工作流程

```
添加微信 → 筛查 → 开会 → 发评估 → 跟进 → 签约
```

## 客服与系统交互要点

1. **加微信后**：先不和系统交互，直接问基本信息做筛查。最多在系统上查看筛查问题清单。
2. **筛查完成后**：不论是否通过，在系统上点击「添加客户」，填写 5 项必填信息（姓名 + 身份截图 + 聊天截图 + 身份选择 + 筛查分类）。
3. **提交后**：后端自动创建学生文件夹、保存聊天记录（含时间戳）、生成学生信息总结。
4. **客户详情页**：可查看信息总结、相关文件、上传更多记录、进度追踪、AI 待办。
5. **权限约束**：客服只能通过界面上传，不能直接修改数据库文件。

## 设计原则

- **按学生归类**：一个学生的所有东西在一个文件夹里——信息总结、逐字稿、聊天记录、评估建议。点击学生名字，一切都在里面。
- **跨学生的放外面**：进度看板、会议安排看板（跨学生视图）、话术库（通用资产）、模板
- **信息总结是活文档**：每次上传聊天记录或会议逐字稿后，AI 自动更新
- **聊天记录按身份分文件**：学生家长聊天记录和学生本人聊天记录分开存储，每条含时间戳
- **客服只上传，不编辑**：客服通过界面上传内容触发系统更新，不能直接修改数据库
