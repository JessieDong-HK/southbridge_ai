## 一、客服工作全流程（7 步）

```
Step 1: 加微信
  家长/学生加微信 → 记录"加微时间" → 计入总加微人数
  ⏰ SLA：12 小时内（最慢 24h）主动发起聊天
      │
      ▼
Step 2: 初次接触 + 筛查
  发送开场话术 → 收集 5 项基本信息 → 判断是否达标
  ├── ❌ 不达标 → 告知风险 → 仅推单模块 → 结束
  └── ✅ 达标 → 发送服务手册 → 截图聊天记录 → 存入系统
      │
      ▼
Step 3: 约会议
  所有达标的人 → 尽量促成免费评估会议
  记录：约了谁、什么时间
      │
      ▼
Step 4: 会议前准备
  根据学生基础情况 → 在我们的发言稿上做个性化调整
  让会议主持人心里有数：这个学生这场会要 cover 什么、怎么说
      │
      ▼
Step 5: 会议
  进行免费评估会议（~30min）
  会议结束后：
  ├── 存储会议记录（逐字稿）→ 同步到学生档案
  └── 发送"评估 & 建议"给学生/家长
      │
      ▼
Step 6: 会议后跟进
  继续同步微信聊天记录（家长/学生提问）
  回答跟进问题
      │
      ▼
Step 7: 促单跟进
  待讨论（具体怎么做一起想）
      │
      ▼
  签约 or 流失
```

---

## 二、Pipeline 阶段定义

每个学生始终处于以下某个阶段：

| 阶段 | 含义 | 触发条件 |
|------|------|---------|
| 新线索 | 加了微信，还未联系 | 加微时自动进入 |
| 初步接触 | 已发开场话术，等待回复 | 客服发了开场 |
| 已筛查 | 收集了 5 项，已判断达标/不达标 | 完成筛查 |
| 已发手册 | 达标，服务手册已发 | 发送手册 |
| 已约会议 | 约好了评估会议时间 | 确认会议时间 |
| 已完成会议 | 评估会议已开完 | 会议结束 |
| 方案沟通中 | 会议后，在谈方案/价格 | 跟进中 |
| 已签约 | 签了 | 🎉 |
| 已流失 | 明确拒绝或长期无回应 | — |

## 三、看板模块设计

### 待办（独立模块）
每天打开工作台，最上方显示今天该做什么。由客服和管理者共同维护。

### 看板（数据仪表盘）
四个核心数字，一眼看清 pipeline 状况：

| 指标 | 来源 |
|------|------|
| 总加微人数 | 所有创建了档案的学生 |
| 达标人数 | status 为"已发手册"及之后阶段的人数 |
| 已约会议 | status 为"已约会议"的人数（含待开会时间） |
| 已签约 | status 为"已签约"的人数 |

### 学生列表
每个学生的卡片，包含阶段、关键信息、最近动态。

## 四、关键业务规则

### SLA
- 加微信后 **12 小时内**（最慢 24h）必须主动发起聊天
- 达标后 **24 小时内** 发送服务手册

### 筛查基线（见 `学生筛选检查清单.md`）
- 学校背景：一线城市头部 / 省市重点
- 校内成绩：前 30-40%
- 语言成绩：雅思 6.5+ / 托福 85+ / 多邻国 110+

### 截图存档节点
- Step 2：筛查聊天截图 → 存入
- Step 6：会议后聊天截图 → 继续追加
- 每次上传按消息级别去重，只追加新的

### 会议相关产出
- 会议前：个性化发言稿（基于学生基础情况调整）
- 会议后：逐字稿存档 + 同步档案 + "评估&建议"发给家长

## 六、公网部署方案

### 架构

```
GitHub 仓库（源代码 + Markdown 数据）
     │
     ├── git push → Vercel 自动部署网站
     │              https://southbridge.vercel.app
     │              Chloe 手机打开 → 看板/待办/学生/上传
     │
     └── 数据通过 GitHub API 读写
          咨询管理/ 下的所有 Markdown 文件
```

### 部署步骤

1. 代码推送到 GitHub
2. Vercel 连接 GitHub 仓库
3. 自动部署 → 获得公网链接
4. Chloe 手机浏览器打开链接即可使用

### 费用

- Vercel：免费（个人项目，月访问量 < 100GB 流量）
- GitHub：免费
- 总成本：¥0/月
