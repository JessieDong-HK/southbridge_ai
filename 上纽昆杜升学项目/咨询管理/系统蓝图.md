## 一、客服标准工作流程（5 步 + 签约）

```
Step 1: 添加微信
  家长/学生加微信 → 客服主动发起聊天
      │
      ▼
Step 2: 筛查
  客服询问基本信息 → 判断学生分类
  ├── 优秀
  ├── 过bar
  └── 不满足
      │
      ▼  （不论筛查结果如何，均需在系统上"添加客户"）
      │
Step 3: 开会
  安排免费评估会议 → 进行会议
      │
      ▼
Step 4: 发评估
  会议结束后 → 发送"评估与建议"给学生/家长
      │
      ▼
Step 5: 跟进
  持续跟进沟通 → 推动决策
      │
      ▼
  签约 or 流失
```

## 二、客服与系统交互 · 完整步骤

### 2.1 加微信后 → 筛查阶段（暂不与系统交互）

- 客服加微信后，**先不和系统交互**
- 客服直接询问对方基本信息，来判断学生是否能通过筛查
- 客服最多在系统上**查看筛查时要问哪些问题**（参考 `学生筛选检查清单.md`）
- 此阶段不需要在系统上做任何录入操作

### 2.2 筛查完成后 → "添加客户"（必须操作）

不管学生有没有通过筛查，客服都应该在系统上点击**「添加客户」**。

**上传界面包含 5 个必填项（全部必填，不填写无法保存）：**

| # | 字段 | 类型 | 说明 |
|---|------|------|------|
| 1 | 学生姓名 | 文本输入 | 填写学生真实姓名 |
| 2 | 微信身份信息页截图 | 图片上传 | 提示：为了提取微信号。截取对方的微信主页（含微信号） |
| 3 | 微信聊天记录截图 | 图片上传 | 提示：请完整截图，确保每一句对话都被截到，重复截到是完全OK的 |
| 4 | 身份选择 | 单选 | 此聊天对象是：**学生家长** / **学生本人** |
| 5 | 筛查分类 | 单选 | 此学生通过筛查发现分类属于：**优秀** / **过bar** / **不满足** |

> ⚠️ 全部 5 项为必填，缺一不可保存此客户。

### 2.3 后端自动处理（客服无需操作）

提交"添加客户"后，后端自动执行以下操作：

**A. 创建学生文件夹结构**

```
咨询管理/[学生姓名]/
├── 会议逐字稿/
│   └── （后续上传的会议逐字稿存放于此）
└── 微信聊天记录/
    ├── 学生本人聊天记录.md    （若身份选择"学生本人"则创建此文件）
    └── 学生家长聊天记录.md    （若身份选择"学生家长"则创建此文件）
```

**B. 保存微信聊天记录**

- 根据客服选择的身份（学生本人 / 学生家长），将聊天记录保存到对应的文件中
- 文件内容包含：**微信号 + 聊天记录**
- 聊天记录保存时附带**时间记录**（x年x月x日x点x分）
- 微信身份信息截图中的微信号被提取后，存入对应的聊天记录文件

**C. 自动生成学生信息总结**

- 系统通过上传的聊天记录，自动生成一份 `学生信息总结.md`
- 此文档是活文档，每次新上传内容后会自动更新

## 四、客户详情页（点击客户名字后进入）

客户详情页分为以下模块：

### 4.1 学生信息总结

- 一个普通的 Markdown 文档
- 由系统通过上传的聊天记录自动生成
- 每次新上传内容后，自动更新此总结
- 界面上不标注"AI 生成"等标签

### 4.2 相关文件

- 显示该学生文件夹里的**所有文件**，有什么就显示什么
- 例如：学生家长微信聊天记录、学生本人微信聊天记录、会议逐字稿、会议总结、评估与建议等
- 所有文件均可**点击查看详情**（均为简单的 Markdown 文档）

### 4.3 上传更多记录

此区域提供两个上传入口：

**A. 上传微信聊天记录截图**
- 选择身份：**学生家长** / **学生本人**
- 上传截图
- 上传后自动追加到对应的聊天记录文件（按时间排序，去重追加）

**B. 上传会议逐字稿**
- 选择参会人（可多选）：
  - 客户方：**学生家长**、**学生本人**
  - 公司方：**Cherie**、**Jessie**、**Chloe**
- 上传文件（支持 TXT 等格式，后端自动转为 `.md` 存储）
- 上传后自动保存到 `会议逐字稿/` 文件夹

### 4.4 进度追踪

清晰显示客服标准工作流程的 5 个阶段，当前所处阶段**重点标出**。

进度信息来自后端数据库：每次新上传内容后，系统自动更新学生信息总结，其中清晰标注当前处于哪个阶段。

**进度显示规则：**

| 阶段 | 完成状态下显示 |
|------|---------------|
| 添加微信 | 显示：学生本人/家长 + 具体添加时间（从聊天记录中获取） |
| 筛查 | 显示：学生分类（优秀 / 过bar / 不满足） |
| 开会 | 显示：会议时间 + 参会人（从会议逐字稿中获取） |
| 发评估 | 显示：评估发送时间 |
| 跟进 | 显示：最近跟进时间 + 跟进状态 |
| 签约 | 🎉 |

- 已完成步骤：显示灰色 + 具体完成信息
- 当前所处阶段：**高亮重点标出**
- 未开始步骤：显示为待定状态

### 4.5 待办显示框

- 系统自动生成的待办事项
- 数据来源：微信聊天记录 + 会议逐字稿
- 每次上传新内容后自动更新
- 客服无需手动编辑
- 界面上不标注"AI 生成"等标签

- **点击学生姓名** → 跳转到该学生的客户详情页
- 看板数据从后端数据库直接读取

## 六、会议安排看板

- 显示：什么时间 + 和谁开会（学生本人/学生家长）
- 数据来源：上传的会议逐字稿中提取的会议时间信息

## 八、Pipeline 阶段定义（与进度追踪对应）

| 阶段 | 含义 | 进入条件 |
|------|------|---------|
| 添加微信 | 已添加微信并开始对话 | 客服上传首个微信聊天记录截图 |
| 筛查 | 已完成基本信息筛查 | 客服完成"添加客户"操作（含筛查分类） |
| 开会 | 已完成评估会议 | 上传会议逐字稿 |
| 发评估 | 已发送评估与建议 | 评估与建议文档已生成并发送 |
| 跟进 | 正在跟进推动决策 | 持续上传新的聊天记录 |
| 已签约 | 签约成功 | 🎉 |
| 已流失 | 明确拒绝或长期无回应 | — |

## 九、筛查分类说明

| 分类 | 含义 | 后续动作 |
|------|------|---------|
| 优秀 | 三项筛查全部达标，且有突出亮点 | 优先推动 → 约会议 |
| 过bar | 三项筛查基本达标 | 正常推进 → 约会议 |
| 不满足 | 任一项不达标 | 诚实告知风险 → 仅推单模块（但仍在系统中建档） |

> 注意：**不满足**的学生同样需要建档，不能跳过"添加客户"步骤。

## 十、权限约束

> ⚠️ **核心原则：不给客服超出描述的额外权限。**

- 客服通过系统界面上传内容（聊天截图、会议逐字稿）
- 客服**不能直接修改**数据库中的 Markdown 文件
- 系统自动生成的内容（学生信息总结、待办事项）由系统自动更新，客服不可手动编辑
- 客服只能通过"上传"这个动作来触发系统更新
- 后台管理员（Jess）保留所有直接编辑权限

## 十一、关键业务规则

### 筛查参考（见 `学生筛选检查清单.md`）
- 学校背景：一线城市头部 / 省市重点
- 校内成绩：前 30-40%
- 语言成绩：雅思 6.5+ / 托福 85+ / 多邻国 110+

### 聊天记录存储规则
- 按身份分文件：学生家长聊天记录、学生本人聊天记录
- 每条记录含时间戳（x年x月x日x点x分）
- 每次上传按消息级别去重，只追加新的
- 重复截图覆盖到的内容完全OK（去重机制处理）

### 会议相关
- 上传逐字稿时必须选择参会人（客户方 + 公司方，可多选）
- 上传后自动保存到会议逐字稿文件夹
- 上传即触发：更新学生信息总结 + 更新进度追踪 + 更新待办事项

## 十五、评估建议自动生成 · 设计

### 定位

评估建议是**客户-facing 文档**，和内部咨询记录不同：

| | 咨询记录（内部） | 评估建议（客户-facing） |
|---|---|---|
| **读者** | 内部（Jess、Chloe） | 外部（家长/学生） |
| **目的** | 全面存档 | 展示专业价值 → 推动决策 |
| **内容** | 讨论点、发言、反思 | 匹配度、时间线、突破方向、方案 |
| **语气** | 中性、全面 | 专业、有温度、可微信直接发送 |

### 存放结构

```
咨询管理/[学生名]/评估与建议/
├── 第1场-评估与建议.md
└── 第2场-评估与建议.md
```

### 文档结构

```markdown
# 评估与建议 · [学生名] · 第X场
> 会议日期 | 参会人

## 学生画像速览
## 匹配度分析（上纽 / 昆杜）
## 申请时间线
## 需要突破的方向
## 推荐方案
## 下一步
```

### 自动化方案：上传即生成闭环

```
Chloe 上传逐字稿（任意格式）
     │
     ▼ 转为 .md 保存到 咨询管理/[学生]/会议逐字稿/
     │
     ▼ 写入触发标记 _trigger_assessment
     │
     ▼ Claude Code 定时扫描（~2min）
     │
     ▼ 自动跑 Workflow → 生成评估建议草稿
     │
     ▼ git commit + push
     │
     ▼ Vercel 自动部署 → Chloe 刷新 → 📋 复制发送
```
