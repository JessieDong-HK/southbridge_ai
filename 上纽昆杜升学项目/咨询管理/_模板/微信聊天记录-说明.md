# 微信聊天记录

> 客服与家长/学生的微信沟通存档。与会议逐字稿并列的客观素材来源。

## 存储规则

- **每个学生一份文件**：`微信聊天记录/[学生名]/完整聊天记录.md`
- **只增不改**：聊天记录只会变长，不会删除或修改已有内容
- **每次上传 = 追加**：客服截图 → AI 提取消息 → 对比已有记录 → 新消息追加上去、旧消息跳过

## 去重机制（消息级别）

每次客服上传截图后：
1. 提取截图中的所有消息（发言人 + 内容 + 时间）
2. 读取该学生已有的 `完整聊天记录.md`
3. **逐条对比**：已在文件中的消息 → 跳过；新消息 → 按时间顺序插入
4. 保存更新后的文件

关键区别：不是"整批跳过"，而是"逐条对比，只加新的"。

## 与档案的关系

每次追加聊天记录后，检查新增消息中是否包含值得更新学生档案的信息（新的成绩、活动、顾虑、决策进展等）。如有 → 更新学生档案。
