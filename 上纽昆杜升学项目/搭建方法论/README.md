# 搭建方法论 — 总览

> 这个文件夹记录的不是"做了什么"，而是"怎么做的"——整个 AI 辅助知识库搭建的完整方法论。

---

## 这是做什么的

我用 AI（Claude Code）搭了一套升学辅导业务的知识库系统，包括：
- **业务知识库**（24个文件）：公司概览、服务项目、课程大纲、案例、品牌调性、SOP
- **外部资源库 — 官网**（9+9个文件）：从680页昆杜官网爬虫 + 2569个上纽官网文件提取整理的专题素材
- **外部资源库 — 公众号**（1+7个文件）：昆杜公众号780篇帖子的内容总览 + 结构化资料库

这个文件夹把整个搭建过程文档化了——每一步的输入、操作、产出、决策、踩坑。

## 这套方法论的适用范围

| 场景 | 是否适用 |
|------|------|
| 有一堆原始资料，想整理成结构化知识库 | ✅ 核心场景 |
| 有一个网站爬虫数据，想提取有用信息 | ✅ 03 直接对应 |
| 想让 AI agent 按需加载文档而非读全部 | ✅ 02 的设计原则 |
| 想持续改进已有知识库的质量 | ✅ 04 的方法 |
| 想批量处理大量同类内容 | ✅ 05 的 prompt 模式 |
| 想在手机上远程控制 Claude Code | ✅ 06 完整教程 |
| 纯技术开发（写代码） | ❌ 不适用，这是内容知识库搭建 |
