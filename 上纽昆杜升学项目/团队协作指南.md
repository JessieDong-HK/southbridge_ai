## 三种角色

```
Owner (Jess)  ← 唯一能改知识库的人，所有产出的审核者
    │
    ├── Builder  ← 搭自动化流程，不能改知识库，产出进 drafts
    │
    └── User    ← 只使用 AI 工具，不改任何文件，不需要 Git
```

| | 👤 Owner | 🤖 Builder | 📱 User |
|---|---------|------------|---------|
| **谁** | Jess | AI 工程师协作者 | 客服、运营、实习生 |
| **改知识库** | ✅ | ❌ | ❌ |
| **写 prompt/workflow** | ✅ | ✅ (在 `collaborator/` 目录) | ❌ |
| **Git** | push main + merge PR | feature 分支 + 提 PR | 不需要 |
| **用什么** | Claude Code + Git | Claude Code + Git | Claude Code 或微信 |

│   ├── jess/                     ← Owner 的工作区

## Builder 手册

### 你能做什么

- ✅ 写 prompt 模板（放在 `prompts/collaborator/`）
- ✅ 搭 workflow / skill / slash command（放在 `workflows/collaborator/`）
- ✅ 用 AI 生成脚本初稿（放在 `outputs/drafts/`）
- ✅ 读取知识库文件作为 AI 的上下文（只读引用）

### 你不能做什么

- ❌ 修改 `公司和辅导产品资料/`、`昆杜官方资料/`、`上纽官方资料/` 下的任何文件
- ❌ 修改 `项目总览.md`、`团队协作指南.md`
- ❌ 修改 `搭建过程记录/` 下的任何文件
- ❌ 直接把产出发布为正式内容（需 Owner 审核）

### 如何加入（首次设置，5 分钟）

```bash
# 1. 克隆仓库
git clone https://github.com/JessieDong-HK/southbridge_ai.git
cd southbridge_ai

# 2. 设置你的 Git 身份
git config user.name "你的名字"
git config user.email "你的邮箱"

# 3. 确认一切正常
git status
# 应该显示：On branch main, nothing to commit

# 4. 看项目总览
# 打开 项目总览.md，了解整体是什么
```

> ⚠️ 如果 `git clone` 提示输入密码，和 Owner 一样需要先创建 GitHub Personal Access Token（见 [GitHub 官方教程](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens)）。

### 日常工作流

```
git checkout main && git pull origin main    # 1. 同步
git checkout -b feature/我的任务              # 2. 开分支
# ... 只在 collaborator/ 和 drafts/ 里工作 ...
git add prompts/collaborator/ workflows/collaborator/
git commit -m "feat: 做什么"                 # 3. 提交
git push origin feature/我的任务             # 4. 推送
→ 在 GitHub 创建 Pull Request               # 5. 提 PR
→ Owner review → merge                      # 6. 等审核
```

## User 手册

### 你能做什么

- ✅ 用 slash command 生成内容（`/generate-video-script` 等）
- ✅ 用 AI 查询知识库（学校信息、服务问题等）
- ✅ 把 AI 产出的内容发布到平台
- ✅ 反馈"生成的不对"给 Owner

### 你不能做什么

- ❌ 修改任何文件
- ❌ 创建或修改 skill / workflow
- ❌ 访问 Git

### 你需要什么

1. Claude Code（或微信）能用
2. 知道有哪些命令可以用
3. 会描述你想要什么

不需要 Git。不需要懂 prompt。不需要看代码。

### 常用命令

| 命令 | 用途 |
|------|------|
| `/generate-video-script` | 输入选题，生成视频脚本 |
| `/generate-xhs-post` | 从视频脚本改写成小红书图文 |
| `/query-knowledge` | 查询学校/服务相关问题 |
| `/customer-faq` | 回答家长常见问题 |

## Git 分支规则

```
main ─────────────────────────────→  (只有 Owner 能 push)
  │
  ├── feature/xxx ──→ PR ──→ Review ──→ merge  (Builder A)
  ├── feature/yyy ──→ PR ──→ Review ──→ merge  (Builder B)
```

| 规则 | 谁 |
|------|-----|
| main 直接 push | 只有 Owner |
| feature 分支 + PR | Builder |
| 知识库文件的 PR 审批 | 只有 Owner 能 approve |

## 常见问题

**Q: Builder 不小心改了知识库文件怎么办？**
→ PR review 时直接拒绝，要求 revert。如果已 merge，Owner 在 main 上 revert。

**Q: User 觉得 AI 产出不对？**
→ 反馈给 Owner。Owner 判断是知识库问题还是 prompt 问题，对应修复后通知 User。

**Q: 多个 Builder 同时工作冲突了？**
→ 各自的 `collaborator/` 目录不重叠，一般不会冲突。真冲突了 Owner 来协调 merge。
