## 一、这个项目是什么

**上海纽约大学和昆山杜克大学在本科申请中都有 AI 面试环节。** 两校的 AI 面试虽然各有自己的题目和打分标准，但整体框架和市面上的求职 AI 面试、其他高校 AI 面试大同小异——本质都是：AI 出题 → 学生回答（语音/文字）→ AI 多维评分 → 反馈报告。

这个项目的目标：**站在巨人的肩膀上，找到一个成熟的开源 AI 面试平台，自定义好两校的题库和评分标准，作为南桥教育的增值服务产品。**

### 业务价值

| 维度 | 说明 |
|------|------|
| **对家长/学生** | 提前模拟真实 AI 面试环境，熟悉流程，降低紧张感，拿到针对性反馈 |
| **对南桥** | 差异化服务产品——市面上没有专门针对上纽/昆杜 AI 面试的模拟平台 |
| **对运营** | 面试过程自动评分，不占用客服/顾问时间；可作为付费增值或免费引流 |

### 与现有系统的关系

```
南桥 AI 自动化工作台
├── 🎬 视频脚本自动化（Phase 1）    ← 内容获客
├── 📋 咨询管理系统（Phase 2）✅      ← 客服转化
├── 🎙️ AI 面试平台（🆕 本项目）      ← 增值服务 / 转化工具
├── 📊 学生进度管理（Phase 3）        ← 服务交付
└── 👥 人员管理（Phase 4）            ← 规模化
```

## 二、为什么不从零搭建

Jess 的核心约束是**时间**。从零搭建一个 AI 面试平台需要：

- 前端 UI（面试界面、语音采集、倒计时、回放）
- 后端服务（STT 语音识别、LLM 追问引擎、TTS 语音合成）
- AI Pipeline（出题策略、追问逻辑、多维评分、报告生成）
- 防作弊、状态管理、并发处理

这些都有成熟的开源方案。**我们的核心工作只有两个：① 选一个最合适的开源项目部署起来；② 自定义题目和评分标准。** 其余全部复用。

### 3.2 重点项目的面试 Workflow

**聆悟 (Lingwu) — 最完整的端到端方案：**

```
HR 输入岗位描述（自然语言）
        ↓
  AI 自动生成面试题 + 评分标准
        ↓
  候选人点击链接进入面试
        ↓
  AI 主持语音/文字面试（含追问）
        ↓
  自动生成多维分析报告
```

**DeepInterview — 语音优先，最接近真实面试体验：**

```
Prep 阶段：CV+JD 分析 → 公司调研 → 出题计划
        ↓
Live 阶段：实时语音面试（STT→LLM→TTS 级联）
        ↓
Post 阶段：评分 + 弱项教练 + 学习建议
```

**structured-hiring — 与南桥现有架构最契合：**

```
create_standard：交互式设计评分标准 → Markdown 文件存储
        ↓
generate_questions：基于标准自动出题
        ↓
evaluate_interview：基于标准评分
        ↓
iterate_standard：人工修改 → AI 学习偏好 → 自动更新标准
```

### 3.3 选型结论（2026-06-23 定）

> **选型：聆悟（yuanzhongqiao/ai-interview-platform）**
>
> 理由：
> - 767 ⭐ / 88 🍴，MIT 协议，社区验证最好
> - Next.js 14 + tRPC + Supabase，和南桥现有技术栈兼容
> - 已深度评估代码：AI 出题引擎、追问引擎、多维度评分报告、tone analysis 全部开箱即用
> - DeepSeek API 可直接驱动（走 OpenAI 兼容接口）
> - 摄像头视频模式已内置，扩展表情/肢体分析仅需 ~100 行增量代码

| 项目 | 淘汰原因 |

## 五、呈现分析引擎（🆕 表情 + 肢体 + 视线）

### 5.0 技术选型

> **选用 `@vladmandic/human`** — 基于 TensorFlow.js 的纯浏览器端人体分析库。
>
> 为什么不用 Hume AI：Hume 的 Expression Measurement API 将于 2026-06-14 关闭（正好是这个月）。
>
> | | @vladmandic/human | Hume AI |
> |------|------|------|
> | 运行位置 | 💻 浏览器本地 | ☁️ 云端 API |
> | 费用 | $0（MIT 开源） | $0.015/分钟 |
> | 隐私 | ✅ 视频不出浏览器 | ❌ 需上传视频 |
> | 可用性 | ✅ 长期可维护 | ⚠️ 本月关闭 |
> | 表情识别 | ✅ 内置 emotion model | ✅ 48 维度专业模型 |
> | 肢体追踪 | ✅ 17 关键点 | ❌ 不支持 |
> | 手势识别 | ✅ 手指追踪 | ❌ 不支持 |

### 5.1 工作原理

```
📹 浏览器摄像头
       │
       ├──→ 聆悟现有：PIP 显示 + 定时截图 + 音频录制
       │
       └──→ 🆕 Human 库（每 2 秒分析一帧，本地 GPU 运算）
              │
              ├── 😊 表情：468 个面部关键点 → 情绪分类
              │         输出：{ happy, neutral, surprised, nervous, ... }
              │
              ├── 🧍 肢体：17 个身体关键点（肩膀/手肘/手腕/髋/膝）
              │         输出：前倾/后仰、是否晃动、手势动作
              │
              └── 👁️ 视线：瞳孔追踪
                        输出：看屏幕 / 看别处 / 看下方
```

### 5.2 与聆悟的集成方案

**集成点 1：面试进行中（voice-interface.tsx）**

```js
// 🆕 新增 hook（~100 行）
const presentation = usePresentationAnalysis({ 
  cameraStream: recording.cameraStream,  // 复用已有摄像头流
  enabled: videoMode,                    // 仅在视频模式下启用
  interval: 2000,                        // 每 2 秒分析一次
});
// 后台静默运行，不干扰面试流程
```

**集成点 2：面试结束时提交**

```js
// voice-interface.tsx 已有的 save 逻辑（line 1220）
// 在原有数据基础上追加一个字段：
fetch("/api/trpc/session.saveRecording", {
  body: JSON.stringify({
    ...原有数据,
    presentationMetrics: presentation.stop(),  // 🆕 表情+肢体汇总
  }),
});
```

**集成点 3：AI 报告生成（summary.ts prompt 增强）**

```
在现有 tone analysis（filler words / hedging / confidence）基础上，
追加视觉分析维度：

"VISUAL ANALYSIS DATA (collected real-time during interview):
 - Emotion timeline: Q1 nervous(0.6)→Q3 confident(0.8)→Q6 relaxed(0.7)
 - Body posture: leaning_forward 80%, upright 15%, leaning_back 5%
 - Gaze: looking_at_screen 90%, looking_away 10%
 
 Based on BOTH the speech transcript AND visual cues,
 assess overall presentation quality..."
```

### 5.3 最终报告效果

```
📊 综合呈现分析

😊 情绪变化
  开场 Q1-Q2：略紧张（nervous 0.6 → 0.3）
  中段 Q3-Q5：自信稳定（confident 0.7-0.8）
  结尾 Q6-Q8：放松自然（relaxed 0.7）

🧍 肢体语言
  整体坐姿前倾（专注度高），无频繁晃动
  Q3 短暂后仰 3 秒（可能对题目不确定）

👁️ 视线方向
  90% 看屏幕，10% 看下方（思考中）

🗣️ 语音分析（聆悟已有）
  filler words: 3 次 "um", 2 次 "嗯"
  整体语速适中，无明显长停顿

🎯 综合呈现评分：7.5/10
  表达流畅，肢体开放，开场略显紧张但快速调整。
```

## 六、题库设计（核心自定义工作）

### 6.1 上纽/昆杜 AI 面试的可能考察维度

> ⚠️ 以下为初步推测维度，实际题库需要基于两校官方资料和真实面试经验来校准。

| 维度 | 说明 | 题型示例 |
|------|------|---------|
| **英语沟通能力** | 全英文面试，评估流利度、表达清晰度 | "Describe a challenge you faced and how you overcame it." |
| **跨文化理解** | 两校均为中美合办，重视跨文化视角 | "How would you bridge cultural differences in a team project?" |
| **学术兴趣与匹配** | 为什么选择这所学校、这个专业 | "Why NYU Shanghai instead of NYU New York?" |
| **批判性思维** | 对开放问题的分析深度 | "Should universities eliminate standardized testing? Why or why not?" |
| **团队协作** | 过往团队经历 + 冲突处理 | "Tell me about a time you disagreed with a teammate." |
| **自我认知** | 对自身优势/劣势的认知 | "What would your classmates say is your greatest weakness?" |
| **时事与价值观** | 对当代议题的看法 | "What global issue concerns you most and what would you do about it?" |

### 6.3 每道题的数据结构

```markdown
---
id: NYUSH-03-001
dimension: 学术兴趣与匹配
difficulty: 中等
time_limit: 120
follow_up_pool:
  - "Can you give a specific example?"
  - "How does that connect to your intended major?"
scoring_rubric: 评分标准/上纽-学术兴趣.md
---

## 题目（显示给学生）

Why did you choose to apply to NYU Shanghai instead of a traditional Chinese university or a university in another country?

## 考察要点（AI 评分参考）

- 对上纽办学模式的理解程度（中美合办 ≠ 普通中外合作）
- 是否有明确的学术/职业目标，且与上纽资源匹配
- 回答中是否体现跨文化意识
- 是否有独立思考，而非背诵官网话术

## 优秀回答特征

- 提到具体的教授、课程、研究项目或海外学习机会
- 展示对上纽"核心课程"（Global Liberal Studies）的了解
- 能解释为什么"在上海 + 属于 NYU 全球网络"对自己重要
- 有个人故事支撑，而非泛泛而谈

## 薄弱回答特征

- 理由空洞（"因为排名高""因为在上海"）
- 与申请其他学校的理由无差异
- 对上纽的了解仅限于百度百科级别
- 无法将学校特点与自身目标关联
```

## 七、评分标准设计

### 7.1 通用评分维度

| 维度 | 权重 | 评分范围 | 说明 |
|------|------|---------|------|
| 内容质量 | 30% | 1-5 | 回答是否切题、有深度、有具体例子 |
| 语言表达 | 25% | 1-5 | 英语流利度、词汇丰富度、语法准确性 |
| 逻辑结构 | 20% | 1-5 | 是否有清晰的结构（观点→论据→例子→总结） |
| 批判性思维 | 15% | 1-5 | 是否展示了多角度思考、独立见解 |
| 临场表现 | 10% | 1-5 | 语速、停顿、自信度（如使用语音面试） |

### 7.2 两校差异化权重

| 维度 | 上纽权重 | 昆杜权重 | 差异原因 |
|------|---------|---------|---------|
| 内容质量 | 30% | 30% | — |
| 语言表达 | 25% | 25% | — |
| 跨文化适配 | +5%（从逻辑拆出） | +5%（从逻辑拆出） | 两校均为中美合办，跨文化是核心考察点 |
| 批判性思维 | 15% | 15% | — |
| 学校匹配度 | 额外加分项 | 额外加分项 | 考察对特定学校的了解深度 |

## 十一、风险与注意事项

| 风险 | 应对 |
|------|------|
| **题库与实际面试差距大** | MVP 后找 1-2 个已录取学生验证，校准题目和评分 |
| **开源项目停止维护** | 优先选 MIT/Apache 2.0 协议的项目，fork 后可自行维护 |
| **学生数据隐私** | 面试数据不入开源项目的数据库；部署时加访问密码 |
| **AI 评分可靠性** | 初期同时看 AI 评分 + 人工抽查，校准后再信任 AI |
| **两校 AI 面试政策变化** | 题库和评分标准存为 Markdown 文件，随时可改 |
